apa pip dan pip-x keduanya bisa digunakan secara bersamaan?
Ya, pip dan pipx keduanya dapat digunakan secara bersamaan dan sebenarnya dirancang untuk saling melengkapi, bukan menggantikan. Mereka melayani tujuan yang berbeda dalam ekosistem Python. 
Perbedaan Tujuan Penggunaan
pip (Package Installer for Python): Digunakan untuk mengelola pustaka dan dependensi proyek Python di dalam lingkungan virtual (virtual environment). Tujuannya adalah untuk memastikan dependensi proyek Anda terisolasi dari proyek lain dan sistem Anda.
pipx: Dirancang khusus untuk menginstal dan menjalankan aplikasi berbasis Command Line Interface (CLI) Python secara global (per pengguna), seolah-olah aplikasi tersebut adalah alat mandiri. pipx secara otomatis membuat lingkungan virtual terisolasi sendiri untuk setiap aplikasi, memastikan tidak ada konflik dependensi antar aplikasi CLI. 
Cara Menggunakannya Bersamaan
Anda dapat menggunakan kedua alat tersebut dalam alur kerja normal Anda:
Gunakan pip untuk menginstal pustaka seperti Django atau requests di dalam lingkungan virtual proyek Anda saat Anda sedang mengembangkan aplikasi.
Gunakan pipx untuk menginstal alat CLI yang ingin Anda akses dari terminal mana pun di sistem Anda, seperti pemformat kode black, linter ruff, atau alat seperti httpie. 
Bahkan, cara yang paling umum untuk menginstal pipx itu sendiri adalah dengan menggunakan pip: 

bash
pip install --user pipx
pipx ensurepath

Singkatnya, pip untuk pustaka proyek, dan pipx untuk aplikasi CLI global Anda. 

sudo apt update
sudo apt full-upgrade
sudo apt update && sudo apt full-upgrade -y
python3 -m pip install --upgrade pip
pip install --upgrade pipx
